AWS S3 入门指南：云存储完全掌握

📅 2026-03-08 👤 Junzhe Lin

Amazon Simple Storage Service (S3) 是 AWS 最受欢迎的对象存储服务之一，以其高可用性、低成本和强大的功能而闻名。本文将帮助你全面了解 S3，并学会构建可靠的对象存储解决方案。

什么是 S3？

S3 是一种对象存储服务，专为从 Web 应用程序大规模存储和检索任意数量的数据而设计。它具有以下核心特性：

  • 无限容量：存储空间无上限，按使用量付费
  • 高可用性：设计可用性为 99.99%
  • 持久性：设计持久性为 99.999999999% (11 个 9)
  • 版本控制：支持对象版本管理，防止误删
  • 生命周期策略：自动将对象转换到更便宜的存储层

核心概念

1. 存储桶 (Bucket)

存储桶是 S3 中用于存储对象的容器。每个存储桶都有一个全局唯一的名称，支持以下操作：

  • 创建和删除存储桶
  • 配置存储桶策略和访问权限
  • 设置跨区域复制
  • 配置生命周期规则

2. 对象 (Object)

对象是存储在 S3 中的基本实体，由以下部分组成：

  • Key：对象的唯一标识符（类似文件路径）
  • Value：实际存储的数据（最大 5TB）
  • Metadata：对象元数据（如 Content-Type, 自定义元数据）
  • Version ID：版本标识符（启用版本控制时）

3. 存储类 (Storage Classes)

S3 提供多种存储类，满足不同的性能和成本需求：

存储类 用途 特点
S3 Standard 频繁访问的数据 高可用性，低延迟
S3 Intelligent-Tiering 访问模式未知的数据 自动优化成本
S3 Standard-IA 低频率访问数据 较低存储成本，检索费较高
S3 Glacier 长期归档 极低成本，检索需几分钟

实战：创建和使用 S3 存储桶

步骤 1：创建存储桶

  1. 登录 AWS 控制台，进入 S3 服务
  2. 点击 "Create bucket" 按钮
  3. 输入全局唯一的存储桶名称
  4. 选择区域（如 us-east-1）
  5. 配置选项（版本控制、加密等）
  6. 设置权限策略

步骤 2：上传对象

可以通过多种方式上传对象：

  • 控制台：直接拖拽文件到存储桶
  • CLI：使用 AWS 命令行工具
  • SDK：通过编程接口上传
# 使用 AWS CLI 上传文件
aws s3 cp myfile.txt s3://my-bucket-name/

# 列出存储桶内容
aws s3 ls s3://my-bucket-name/

# 下载文件
aws s3 cp s3://my-bucket-name/myfile.txt ./

步骤 3：设置访问权限

S3 提供多种访问控制方式：

  • IAM 策略：精细化的权限控制
  • 存储桶策略：基于 JSON 的策略语言
  • 访问控制列表 (ACL)：细粒度的对象级权限

示例：存储桶策略允许公开读取

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::my-bucket-name/*"
        }
    ]
}

版本控制

启用版本控制后，S3 会保留对象的每个版本，防止意外删除或覆盖：

  • 可以恢复对象的任何历史版本
  • 删除对象时，只是添加删除标记，原版本仍可恢复
  • 适用于需要数据审计和回滚的场景

生命周期策略

通过生命周期规则，可以自动将对象转换到更便宜的存储类：

# 示例：30天后转为 Standard-IA，90天后转为 Glacier
规则 1：创建 30 天后 → S3 Standard-IA
规则 2：创建 90 天后 → S3 Glacier
规则 3：创建 365 天后 → 删除

安全最佳实践

保护 S3 数据安全是每个云工程师的职责：

  • 默认情况下，存储桶应设为私有
  • 启用版本控制防止数据丢失
  • 使用加密保护静态数据（SSE-S3 或 SSE-KMS）
  • 启用 CloudTrail 审计日志
  • 使用 MFA Delete 防止意外删除
  • 定期审查访问策略，避免公开敏感数据

总结

AWS S3 是云原生应用的重要基础设施，掌握 S3 的使用方法对于现代开发至关重要。通过本文，你应该已经了解了 S3 的核心概念、存储类选择、访问控制以及最佳实践。

S3 的功能远不止于此，你还可以探索诸如跨区域复制、事件通知、静态网站托管等高级功能。
